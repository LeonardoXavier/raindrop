Guiding Principles for Raindrop

Raindrop is an experiment in open messaging uses the following guidelines for its development.

Raindrop should be:

Design driven

  • A great, targeted user experience is the ultimate goal. An extensible platform will be used for the experience, but the platform is being used to service the user experience. A general extensible platform is not the end goal.
  • Community feedback on the user experience is important, but the experience will not be designed by committee.

Web-based

  • HTML, Javascript, CSS and HTTP are the basic building blocks of the experience.
  • For the user experience, a modern browser's capabilities should be assumed, like the capabilities in Firefox 3.5+, Safari 4 and Chrome 3+.

Secure

  • Your messages and accounts should be properly secured.
  • Protections around extensions should be in place to make the extensions safe and enjoyable to use.

Complementary

  • The Raindrop platform collects conversations that are important to you from around the web, from existing message services (email, blogs, twitter, social networks).
  • The platform's initial focus is to work with the existing services, and not become a messaging service in and of itself.

Participatory

  • Raindrop users should have proper support channels, not using a bug tracking system to suggest new things.
  • Anyone how knows web technologies should be able to extend Raindrop and share that extension with others.
